"Active contours"  About page.

This GUI was written as a part of the “Medical Imaging” course. Together with Alex Blekhman (http://www.mathworks.com/matlabcentral/fileexchange/authors/26540)- my partner we have implemented the famous paper "Active Contours without Edges" by Tony Chan and Luminita Vese.  The paper implementation was wrapped up with a GUI to allow easy parameters change and segmentation process view and results storage for future analysis.
Later on we have added two existing implementation of additional methods to compare between the methods, and to learn each. The method are:
1) Implementation of "Level Set Evolution Without Re-initialization: A New Variational Formulation" in Proceedings of CVPR'05, vol. 1, pp. 430-436.  Author: Chunming Li, all rights reserved.
    E-mail: li_chunming@hotmail.com
    URL: http://www.engr.uconn.edu/~cmli/
2) Implementation of "Localizing Region Based Active Contours" in “Image Processing, IEEE Transactions on “ im Nov. 2008 by Lankton and Tannenbaum.  Coded by: Shawn Lankton (www.shawnlankton.com)

This is the place to thank both authors for the code contribution.

The users who download this code are welcome to play with it, to try and learn the above segmentation schemes. Moreover, additional methods can be added, and the GUI may serve as a convenient platform for segmentation methods understanding, analysis and comparisons.
Best segmenting you all!

Changes list:

1.04

  • Color space panel added

    1. User can choose the color space in which the contours segmentation scheme will be tested.

    2. As each color space consists of several dimensions, the user must choose the color dimension most suitable for his application. Here, Y in YCbCr is best for segmentation of object with contrast different form his surrounding, while Cb/Cr are good to find object with color different form its surrounding, disregarding the illumination dependent contrast.

  • Blur (denoise) panel added

    1. Blur is created using NxN Gaussian window according to user defined sigma. Use if blur demonstrates both, how deblurring deals with noise, and how it effects the contours segmentation scheme, allowing the user to choose the appropriate denoisng parameters.

    2. sigma varies between [0:100]%. 0 value will result in no blur at all, 100 value will result in Box-blur. Higher sigma value results in higher blur.

    3. N varies between 4 and minimal image dimensions. Higher N value results in higher blur.

  • Mask definition tool changed

    1. Freehand selection mode is supported (instead of polygon).

    2. Any number of regions can be selected.
